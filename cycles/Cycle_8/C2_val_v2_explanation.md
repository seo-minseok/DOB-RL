# C2_val_v2.py 코드 설명

## 전체 목적

> **"학습된 모델이 미래를 예측할 때, 어디서/언제 신뢰를 잃는가?"** 를 시각화

학습이 완료된 DOB-MBRL 모델을 검증하는 시각화 도구.  
"모델 기반 롤아웃에서 불확실성이 어떻게 작동하는지"를 영상으로 보여준다.

---

## 동작 흐름

### 1단계 — 롤아웃 수집 (실제 시뮬레이션 없이 모델만으로)

```
초기 상태 obs0 (10개, 노이즈 추가)
        ↓
  정책(QNetwork)으로 행동 선택
        ↓
  NormalizedRBF로 불확실성 측정
        ↓
  불확실성 < THRESHOLD(0.1)?  ──No──→ 해당 롤아웃 중단 (stop_steps 기록)
        ↓ Yes
  ResidualDxNet + 명목 동역학으로 다음 상태 예측
        ↓
  (반복, 최대 10 스텝)
```

10개의 롤아웃을 동시에 진행하되, **각 롤아웃이 언제 불확실성 임계값을 초과하는지** 기록한다.  
이것이 C2(Criterion 2), 즉 **적응형 롤아웃 절단 기준**이다.

---

### 2단계 — 프레임별 비디오 생성

각 스텝마다 두 개의 패널로 구성된 프레임을 생성한다.

#### 왼쪽 패널 — 2D 상태 공간 + 불확실성 히트맵

```
          theta(pole angle)
    ┌──────────────────────────┐
    │  [히트맵: 불확실성 크기]   │  ← 현재 hero의 속도/각속도를 조건으로 계산
    │                          │     파란색=낮음, 빨간색=높음
    │  ──○ hero trajectory     │  ← rollout 0번 (굵은 파란선)
    │  ── grey trajectories    │  ← rollout 1~9 (얇은 회색선)
    │  × = 중단된 지점          │
    └──────────────────────────┘
              x(cart position)
```

히트맵은 단순히 하나의 경로만 보는 게 아니라, `(x, theta)` 전체 공간에서  
"지금 hero의 v, omega, action 조건에서 어느 영역이 불확실한지"를 보여준다.  
즉 **불확실성의 지형도**다.

#### 오른쪽 패널 — 시간에 따른 불확실성 크기

```
uncertainty
    │         × ← 이 지점에서 롤아웃 중단
    │    /\  
    │   /  \──×
0.1 │─────────────────── (THRESHOLD 빨간 선)
    │──/
    └──────────────────── step
```

각 롤아웃의 불확실성이 시간에 따라 어떻게 변하고,  
어느 스텝에서 임계값을 넘어 중단되는지 보여준다.

---

## Hero란?

`HERO_IDX = 0`으로 지정된 **0번 롤아웃**이다.  
이 롤아웃의 현재 속도(`v`)와 각속도(`omega`)를 **조건값으로 고정**하여 히트맵을 계산한다.

> "지금 hero가 경험하고 있는 동역학 조건에서, 상태 공간 전체의 불확실성 분포가 어떤가?"

---

## 핵심 구성요소

| 구성요소 | 역할 |
|---|---|
| `QNetwork` | 각 상태에서 최적 행동 선택 |
| `ResidualDxNet` | 명목 모델이 놓친 잔차 동역학 예측 |
| `NormalizedRBFModel` | 현재 (상태, 행동)에서 모델의 불확실성 추정 |
| `THRESHOLD = 0.1` | 불확실성이 이를 초과하면 롤아웃 신뢰 불가로 판단, 중단 |
| `predict_next_obs_dob` | 명목 동역학 + 잔차 보정으로 다음 상태 예측 |

---

## 주요 파라미터

| 파라미터 | 값 | 설명 |
|---|---|---|
| `NUM_ROLLOUTS` | 10 | 동시에 진행하는 롤아웃 수 |
| `MAX_HORIZON` | 10 | 롤아웃 최대 스텝 수 (`config.max_horizon_length` 기준) |
| `THRESHOLD` | 0.1 | 불확실성 절단 임계값 (`config.uncertainty_threshold` 기준) |
| `SEED` | 1 | 불러올 체크포인트 시드 번호 (`Champion_Seed{SEED}_BestModel.pt`) |
| `HERO_IDX` | 0 | 히트맵 조건으로 사용할 롤아웃 인덱스 |
| `FPS` | 4 | 출력 영상 프레임 속도 |

---

## 출력

`C2_Batch_Adaptive_Rollout_Hero.mp4` — 각 스텝별 불확실성 히트맵 + 롤아웃 경로 + 불확실성 그래프가 담긴 영상
