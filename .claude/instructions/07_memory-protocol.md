# 07 — 메모리 업데이트 프로토콜

## 읽기 (모든 작업 시작 전 필수)

모든 작업 시작 시 `memory.md`를 반드시 먼저 읽어라.
여기에 기록된 내용은 이전 Cycle에서 발견된 실수, 주의사항, 패턴이다.

## 쓰기 프로토콜

1. **에이전트가 제안**: Cycle 완료 후 memory.md에 추가할 후보 항목을 대화에서 제시
2. **사용자가 승인 또는 거부**
   - 승인 → `memory.md`에 항목 추가
   - 거부 → 스킵. **동일 항목 재제안 없음**

## memory.md 수정 금지 케이스

- 사용자 승인 없이 memory.md 자동 수정 금지
- 거부된 제안의 재제안 금지

## memory.md 구조

```markdown
## 반복 실수 방지
- 항목

## 환경/도구 관련
- 항목

## 코드 패턴
- 항목

## 실험 교훈
- (Cycle별로 승인된 교훈 누적)
```

## 제안 품질 기준

- 구체적이고 재현 가능한 교훈만 제안
- "이번에 실수했음" 수준이 아닌, 다음 Cycle에서 적용 가능한 규칙 형태로
- 예: "matplotlib subplot 사용 시 반드시 (1,1) 확인 — 여러 metric을 한 figure에 그리다가 규칙 위반 발생"
